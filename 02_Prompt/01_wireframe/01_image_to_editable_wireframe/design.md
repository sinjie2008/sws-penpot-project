# 01 Image-to-Editable-Wireframe / design.md

## Purpose

This file defines the low-fidelity wireframe design direction for converting a source image into an editable Penpot wireframe.

Target Penpot page:

```text
01 Wireframe
```

This stage is for structure only.
This is not final design.

---

## Visual style

Use:

```text
black and white only
grayscale only when needed
pure white page background
black text
simple outlines
simple dividers
simple placeholder boxes
```

Do not use:

```text
Superworld brand colors
final color palette
gradients
shadows
photos
image fills
icons with final styling
polished card design
decorative background
```

---

## Image rule

Every image or photo area from the source image must become a placeholder box.

Use labels such as:

```text
Image
Hero Image
Product Image
Application Image
Thumbnail
Banner
News Thumbnail
```

Do not use the original image inside the wireframe unless the user explicitly asks.

---

## Text rule

All visible text should be editable.

Keep these as separate text layers:

```text
heading
subheading
paragraph
button text
label
caption
tag
card title
list item
table text
navigation item
news title
date
category
```

Do not combine unrelated text.

Preserve source text exactly:

```text
wording
spelling
capitalization
punctuation
symbols
navigation names
labels
dates
categories
```

Do not paraphrase.
Do not simplify.
Do not correct grammar.
Do not invent missing content.

If text is unclear, mark:

```text
UNCERTAIN: [approximate section]
```

---

## Element rule

All visible elements should be separately editable.

Separate:

```text
section containers
cards
buttons
image placeholders
dividers
icons
labels
tables
rows
columns
navigation items
```

Do not flatten the page.

---

## Layout rule

Default desktop wireframe:

```text
frame width: 1440 px
content width: 1200 px to 1280 px
background: pure white
content alignment: centered
```

Use 8-point spacing system.

Preferred spacing:

```text
4
8
12
16
24
32
40
48
64
80
96
```

---

## Output expectation

The completed wireframe should be:

```text
low-fidelity
clean
easy to understand
editable
separated into clear sections
aligned to consistent spacing
ready for user review before final design
```
