# 01 Wireframe / wireframe-prd.md

## Product requirement
Create a low-fidelity Penpot wireframe from the provided input files.

Target Penpot page:

`01 Wireframe`

---

## Objective
The wireframe should help the user review:

1. page structure
2. content order
3. section hierarchy
4. layout balance
5. navigation flow
6. CTA position
7. image/content placement

This is not final design.

---

## Input source
Use files from:

- `01_Input/raw-wireframe/`
- `01_Input/reference-image/`
- `01_Input/content-notes/`
- `01_Input/product-info/`

If a raw wireframe image is provided, convert it into editable Penpot structure.

If content notes are provided, use them to label sections and text blocks.

If product information is missing, use simple placeholders.

---

## Target frame
Default desktop frame:

- width: `1440 px`
- content width: `1200 px` to `1280 px`
- background: pure white

Do not create mobile/tablet frame unless the user provides size and approval.

---

## Penpot page
Create or update the frame inside:

`01 Wireframe`

Do not update other Penpot pages.

---

## Recommended frame name
Use this naming pattern:

`[Page Name] / Wireframe / Desktop`

Example:

`Homepage / Wireframe / Desktop`

If the page name is unknown, use:

`Website Page / Wireframe / Desktop`

---

## Required wireframe sections
Use the sections required by the actual page.

If no section list is provided, use a safe website page structure:

1. Header
2. Hero
3. Intro / Overview
4. Main Content Section
5. Supporting Content Section
6. CTA Section
7. Footer

Do not add unnecessary sections.

---

## Header requirement
Show:

- logo placeholder
- main navigation
- optional search/contact button
- clear spacing

Use black-and-white structure only.

---

## Hero requirement
Show:

- page title
- short intro text
- primary CTA
- secondary CTA if needed
- hero image placeholder if needed

Do not use final image styling.

---

## Content section requirement
For each section:

- include section title
- include short supporting text if provided
- use cards, rows, tabs, or grids only when useful
- keep the layout clean and editable

---

## CTA requirement
CTA should be visible but simple.

Examples:

- `View Products`
- `Contact Us`
- `Send Enquiry`
- `Learn More`

Do not style as final button design.

---

## Footer requirement
Use simple footer structure.

Show:

- logo placeholder
- link groups
- contact area
- copyright row

Only include if footer is part of the requested page.

---

## Wireframe visual requirement
Use:

- black text
- white background
- grayscale lines
- simple boxes
- simple outlines
- no decorative styling

Do not use:

- color
- gradient
- shadow
- blur
- photo
- illustration
- polished design effect

---

## Editability requirement
Every visible item must remain editable.

Keep separate:

- text
- cards
- placeholders
- dividers
- buttons
- icons
- section containers

Do not flatten.

---

## Success criteria
The wireframe is successful when:

- it matches the intended page structure
- it is easy to review
- it is not high-fidelity
- all important content is included
- all objects remain editable
- spacing is consistent
- no wrong Penpot page is changed
- all errors are logged
