## Wireframe goal
Convert the raw input into a clean, editable Penpot wireframe.

The wireframe must show:

- page structure
- section order
- content hierarchy
- major layout blocks
- navigation placement
- CTA placement
- image placeholder placement
- card/list/table placement where needed

The wireframe must not look like final design.

---

## Visual style
Use:

- black and white only
- grayscale only when needed
- pure white page background
- black text
- simple outlines
- simple dividers
- simple placeholder boxes

Do not use:

- Superworld brand color
- final color palette
- gradients
- shadows
- photos
- image fills
- icons with final styling
- polished card design
- decorative background

---

## Image rule
Every image from the reference must become a placeholder box.

Use labels such as:

- `Image`
- `Hero Image`
- `Product Image`
- `Application Image`
- `Thumbnail`
- `Banner`

Do not use the original image inside the wireframe unless the user explicitly asks.

---

## Text rule
All visible text should be editable.

Keep these as separate text layers:

- heading
- subheading
- paragraph
- button text
- label
- caption
- tag
- card title
- list item
- table text

Do not combine unrelated text.

---

## Element rule
All visible elements should be separately editable.

Separate:

- section containers
- cards
- buttons
- image placeholders
- dividers
- icons
- labels
- tables
- rows
- columns
- navigation items

Do not flatten the page.

---

## Layout rule
Default desktop wireframe:

- frame width: `1440 px`
- content width: `1200 px` to `1280 px`
- content centered
- background: pure white

Use 8-point spacing system.

Preferred spacing:

- 4
- 8
- 12
- 16
- 24
- 32
- 40
- 48
- 64
- 80
- 96

## Output expectation
The completed wireframe should be:

- low-fidelity
- clean
- easy to understand
- editable
- separated into clear sections
- aligned to consistent spacing
- ready for user review before final design

---

## Completion checklist
Before finishing, confirm:

- all important visible text is included
- all major sections are included
- all image areas are placeholders
- all text is editable
- all objects are separately editable
- no color styling is used
- no final design styling is used
- spacing is consistent
- correct Penpot page was updated
- `05_Log_Tracking/mcp-log.md` is updated
- `05_Log_Tracking/error-log.md` is updated if any error happened
- `03_Penpot_Status/status.md` is updated
