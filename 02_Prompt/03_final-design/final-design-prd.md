# 03 Final Design / final-design-prd.md

## Product requirement
Create a high-fidelity desktop final design in Penpot based on the approved wireframe.

Target Penpot page:

`03 Final Design`

---

## Important note
This file is for final design.

Do not name this file `wireframe-prd.md` because that will confuse the final design stage with the wireframe stage.

---

## Objective
The final design should help the user review:

1. final desktop visual direction
2. typography hierarchy
3. component styling
4. section spacing
5. product/application content presentation
6. CTA clarity
7. overall B2B credibility

---

## Required source
Use:

- approved wireframe from `01 Wireframe`
- design rules from `00_Rules/design-rules.md`
- content from `01_Input/content-notes/`
- product information from `01_Input/product-info/`
- approved reference images from `01_Input/reference-image/`
- components from `02 Components / Design System`

Do not design from raw input only if an approved wireframe already exists.

---

## Target frame
Default desktop frame:

- width: `1440 px`
- content width: `1200 px` to `1280 px`
- layout: centered
- background: follow approved design direction

Do not create mobile layout in this stage.

---

## Penpot page
Create or update the frame inside:

`03 Final Design`

Do not update other pages.

---

## Recommended frame name
Use this naming pattern:

`[Page Name] / Final Design / Desktop`

Example:

`Homepage / Final Design / Desktop`

If the page name is unknown, use:

`Website Page / Final Design / Desktop`

---

## Required design treatment
Apply:

- modern B2B design
- clean section separation
- consistent grid
- controlled whitespace
- clear typography hierarchy
- clear CTA hierarchy
- reusable components
- professional product/application presentation

---

## Header requirement
Header should include:

- Superworld Electronics logo area
- navigation
- contact/enquiry action
- clean desktop alignment

Use the approved header component if available.

---

## Hero requirement
Hero should include:

- strong page title
- short supporting copy
- primary CTA
- secondary CTA if useful
- image or visual area if approved

Keep text concise.

---

## Section requirement
Each section should have:

- clear section heading
- short purpose text
- scannable content
- consistent spacing
- clear relation to user goal

Avoid repeated meaning across sections.

---

## Card requirement
Cards should be:

- consistent
- aligned
- readable
- not too text-heavy
- easy to scan

Card content should not become long paragraph blocks.

---

## Product/application content requirement
When product or application content is included:

- keep component family and product series clear
- show relationship between application area and component type
- do not invent unavailable series
- do not invent specifications
- use placeholders if information is missing

---

## CTA requirement
CTA should clearly guide the user.

Possible CTA labels:

- `View Products`
- `Explore Applications`
- `Send Enquiry`
- `Request Sample`
- `Contact Us`
- `Download Catalogue`

Use only CTAs suitable for the actual page.

---

## Footer requirement
Use approved footer component if available.

Footer may include:

- logo
- company links
- product links
- application links
- contact information
- copyright

---

## Style requirement
Use:

- Poppins for headings
- Inter for body text
- consistent spacing scale
- approved component styles
- controlled color usage

Do not use random one-off styles.

---

## Success criteria
The final design is successful when:

- it is clearly based on the approved wireframe
- it looks modern and B2B
- it is easy to scan
- content is concise
- components are consistent
- spacing is consistent
- no wrong page is edited
- all errors are logged
- status is updated
