# 03 Final Design / codex-goal-command.md

## Goal command
Use this command when asking Codex or the agent to create the final desktop design in Penpot through Penpot MCP.

---

## Command

Read and follow these files:

1. `00_Rules/AGENTS.md`
2. `00_Rules/design-rules.md`
3. `03_Penpot_Status/status.md`
4. `02_Prompt/03_final-design/design.md`
5. `02_Prompt/03_final-design/final-design-prd.md`

Use source material from:

- approved Penpot page `01 Wireframe`
- Penpot page `02 Components / Design System`
- `01_Input/reference-image/`
- `01_Input/content-notes/`
- `01_Input/product-info/`

Use Penpot MCP only.

Target Penpot page:

`03 Final Design`

Task:

Create or update the approved desktop final design frame based on the approved wireframe and final design instruction files.

Rules:

- Confirm Penpot MCP is available before editing.
- Confirm the active Penpot project/file before editing.
- Work only on Penpot page `03 Final Design`.
- Do not update `01 Wireframe`.
- Do not update `02 Components / Design System` unless component creation is explicitly approved.
- Do not update `04 Mobile Design`.
- Do not update `05 Prototype / Flow`.
- Use the approved wireframe as structure.
- Use components from `02 Components / Design System` where available.
- Use Poppins for headings.
- Use Inter for body text.
- Use consistent 8-point spacing.
- Use content width `1200 px` to `1280 px`.
- Use desktop frame width `1440 px` unless another approved size is provided.
- Keep the design modern, clean, professional, and B2B.
- Avoid long paragraphs.
- Avoid duplicated sections.
- Do not copy competitor content.
- Do not invent product specifications.
- Use real images only if provided and approved.
- If images are missing, use clean placeholders.
- Keep text editable.
- Keep objects separately editable.

Logging requirement:

- Add a new entry to `05_Log_Tracking/mcp-log.md`.
- Add a new entry to `05_Log_Tracking/design-change-log.md`.
- If any issue happens, add a new entry to `05_Log_Tracking/error-log.md`.
- If any issue is fixed, add a new entry to `05_Log_Tracking/fixed-issue-log.md`.
- Update `03_Penpot_Status/status.md` after completion.

After completion, report:

1. Penpot page updated
2. Frame updated or created
3. Wireframe source used
4. Components used
5. Input files used
6. Exact sections designed
7. Any issue found
8. Log files updated
9. Next step
