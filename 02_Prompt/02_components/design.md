# Design Rules — Components

## Typography

Use only these fonts:

```text
Headings: Poppins
Body: Inter
```

Recommended typography styles:

```text
Display / Hero Title
H1
H2
H3
Body Large
Body Regular
Body Small
Caption
Button Text
Navigation Text
```

## Default Color Palette

Before using these colors, Codex must ask user to confirm.

```text
Primary Green: #3ec272
Secondary Blue: #4077be
Navy: #203362
Red: #ee3036
Yellow: #fdb740
Black: #000000
White: #ffffff
```

## Color Usage Rule

```text
60% White
30% Green #3ec272
10% Blue #4077be
```

## Spacing

Codex must scan spacing from `01 Wireframe` first.

Recommended spacing token names:

```text
spacing-xs
spacing-sm
spacing-md
spacing-lg
spacing-xl
section-padding
card-padding
button-padding
grid-gap
```

## Style Direction

Use a clean, modern corporate B2B style.

Design direction:

```text
Clean
Professional
Modern
Corporate
Apple-like spacing
No unnecessary decoration
No random colors
No messy layout
```

## Border Radius

Use no border radius unless approved.

```text
Default radius: 0px
```

## Component Naming

Use clear names:

```text
Button / Primary / Default
Button / Primary / Hover
Button / Secondary / Default
Button / Outline / Default
Button / Disabled
Navigation / Header / Desktop
Navigation / Header / Mobile
Navigation / Breadcrumb / Default
Navigation / Sub Navigation / Default
Card / Product / Default
Card / Application / Default
Card / News / Default
Section / Hero / Default
Section / Title Block / Default
Section / Image Text / Default
Section / CTA / Default
Form / Input / Default
Form / Input / Error
Footer / Default
```

Do not use names like:

```text
Group 1
Rectangle 25
Text copy
Frame 123
Untitled component
```

## Layout Rules

1. Keep all components aligned properly.
2. Keep spacing consistent.
3. Keep component size reusable.
4. Do not over-design.
5. Do not add decoration that is not required.
6. Do not change the original wireframe.
7. Use clear section labels inside `02 Components`.
